This repository contains a collection of imformation retrieval implementations for fundamental search problems, developed as part of the Information Retrieval Course at the **Athens University of Economics and Business (AUEB), Department of Informatics.**

## Team Project

## Phase 1: Baseline Information Retrieval with ElasticSearch

This phase focuses on implementing a classical retrieval system using ElasticSearch to perform lexical matching on the **IR2025** collection. The goal is to establish a performance baseline using traditional statistical models.
### Implementation Overview

The system follows the standard retrieval pipeline where documents are indexed and then matched against user queries using a mathematical ranking function.

**1. Data Preprocessing**

* The **IR2025** collection, consisting of 18,316 documents (`documents.csv`), was processed for compatibility with ElasticSearch.

* Queries from `queries.csv` were cleaned and formatted for batch execution.

**2. Indexing and Analyzer Configuration**

* A specialized **Inverted Index** was created to store the document collection.

* The system utilizes a custom **Analyzer** to perform tokenization and filtering (e.g., stop-word removal, stemming).

* The **BM25 (Best Matching 25)** similarity function was selected as the primary ranking model.

**3. Retrieval Process**

* Queries are transformed into clauses and executed against the inverted index.

* For each query, the system retrieves the top **k** results (k=20,30,50) based on their similarity scores.

* The ranking function calculates scores based on term frequency and document frequency within the collection.

## Evaluation

The effectiveness of the baseline system is measured using the trec_eval tool by comparing retrieved results against the ground truth (`qrels.txt`).

## Performance Metrics

* **MAP (Mean Average Precision):** Evaluates the overall quality of the ranked list.

* **Precision@k:** Measures the accuracy within the first k=5,10,15,20 retrieved documents.

## Evaluation Tool

The performance of the retrieval system is evaluated using **trec_eval**, the standard tool used by the Text REtrieval Conference (TREC) community.

### Installation

Since this repository does not include the trec_eval binaries (executables), you need to download and compile it:

* Official Source: Download the latest version from NIST trec_eval or clone the official GitHub repository:
```bash
git clone https://github.com/usnistgov/trec_eval.git
```
Compilation:
Navigate to the folder and run:
```bash
make
```

(Note: On Windows, you might need a C compiler like MinGW or use the provided binaries from your course material).

### Usage

To evaluate your results, use the following command format:
```bash
./trec_eval -q qrels.txt results.txt > trec_eval_results.txt
```

* -q: Detailed evaluation for each query.
* qrels.txt: The ground truth relevance judgments.
* results.txt: The output file generated by your search engine.
* trec_eval_results.txt: The output file generated by trec_eval
